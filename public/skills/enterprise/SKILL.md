        ## Built-in Compliance Features

HTS tokens have protocol-level compliance keys that don't require smart contract code:

        ### KYC Key

Grant or revoke KYC status for specific accounts. Only KYC-approved accounts can hold or transfer the token. **Use case:** Securities tokens, regulated financial instruments.

        ### Freeze Key

Freeze a specific account's token holdings — they can't send or receive. **Use case:** Suspicious activity, regulatory holds.

        ### Wipe Key

Remove tokens from a specific account (clawback). **Use case:** Court orders, error correction.

        ### Pause Key

Pause ALL operations on the token globally. **Use case:** Emergency circuit breaker during exploits.



**These compliance features are unique to Hedera at the protocol level.** On Ethereum, each of these requires custom Solidity code, which means each implementation can have different bugs. On Hedera, they're standardized, audited, and built into the consensus layer.
